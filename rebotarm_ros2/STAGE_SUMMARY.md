# reBotArm ROS 2 阶段性总结

## 当前定位

当前工作空间是面向六轴 DM 电机机械臂的可移动重构版本。整个
`rebotarm_ros2` 目录移动后，可以在新位置重新构建和运行；源码与配置不依赖
原机器的项目绝对路径，也不依赖启动命令所在的当前目录。

## 已完成能力

- DM-USB2FDCAN、SocketCAN `can0`、1 Mbps 通信链路
- 六个 DM 电机的连接生命周期、完整反馈等待和故障传播
- `/joint_states`、单关节状态、机械臂诊断和完整 URDF/TF/RViz
- 默认锁定的使能、安全回零、标零和关节轨迹接口
- 当前目标预装载、最小加加速度轨迹、速度/步长/关节限位
- 路径容差、目标容差、目标时间容差、取消保持和异常自动失能
- 标准 `FollowJointTrajectory` 接口及完整 desired/actual/error/time 反馈
- MoveIt 2 的 KDL、OMPL、SRDF 和控制器映射
- 假机 MoveIt 规划到控制器执行的完整闭环

## 已完成真机验证

- 只读六电机反馈、使能后当前位置保持、安全回零和失能
- `joint6: 0 → 0.02 → 0 rad` 受限位置运动
- `0.02 rad` 目标的实测位置为 `0.020409 rad`
- 返回零位实测位置为 `0.000191 rad`
- 已执行测试中 CAN 错误、丢包和 bus-off 均为零

## 安全边界

- 电机没有抱闸，失能后姿态下垂不是零位丢失。
- 当前机械位置不能重新写成零位；`set_zero` 只用于机械零位校准维护。
- 运动、回零、校准默认全部锁定，必须通过启动参数明确打开。
- 轨迹错误会自动失能；正常轨迹结束后保持末点，直到显式失能。
- `scripts/scan_motors.sh` 只检查接口并扫描 1–7 号电机，不会修改或启动 CAN 接口。

## 当前验证状态

- 全工作空间四个 ROS 2 包可以完整构建。
- 自动测试：78 项通过，0 项失败。
- MoveIt 假机规划、轨迹下发、反馈和执行结果已闭环通过。

## 下一阶段

下一项必须依赖真机的验证，是由 MoveIt 规划并执行一次小幅多路点轨迹，随后
返回标定零位并失能。通过后再考虑笛卡尔目标、重力补偿、低层指令和末端夹爪；
这些功能不能仅凭假机结果宣布可用。

## 脚本入口

所有脚本均可从任意当前目录调用，并以脚本自身位置定位工作空间：

```bash
scripts/build.sh
scripts/scan_motors.sh can0
scripts/start_real.sh can0
# 保持上一条命令运行，在另一个终端执行：
scripts/safe_home.sh
```

`start_real.sh` 会自动加载 ROS 2 Humble 和本工作空间的 `install/setup.bash`；
首次运行没有 `install` 时会先构建。它启动真实 DM 驱动，并只解锁安全回零服务。
`safe_home.sh` 最多等待服务 10 秒，随后要求人工输入 `yes` 确认机械臂周围安全。
